迷你 Transformer 训练学习笔记

基于 Tiny Shakespeare，从零训练一个字符级 Mini GPT。目标不是得到可用大模型， 而是把大模型训练里最核心的链路跑通：数据、Embedding、位置编码、Self-Attention、 Causal Mask、Cross Entropy、反向传播、loss 下降和文本生成。

1. 学习目标

这次实验训练的是一个很小的 Transformer，参数量约 0.83M， 数据集是 Tiny Shakespeare，任务是字符级 next-token prediction。

  • 理解语言模型训练本质：给定前文，预测下一个 token。
  • 理解 Transformer 内部：Embedding、位置编码、Self-Attention、MLP、残差连接、LayerNorm。
  • 理解 Causal Mask：训练自回归语言模型时不能偷看未来 token。
  • 理解 Cross Entropy Loss：比较模型预测分布和真实下一个字符。
  • 理解反向传播：loss.backward() 计算梯度，optimizer.step() 更新参数。
  • 观察 loss 下降：从随机预测逐步学到莎士比亚文本的统计规律。

这个实验不是为了工作中直接使用模型，而是为了能讲清楚“大模型训练到底发生了什么”。

2. 训练主流程

训练流程可以概括为下面这条链路：

文本数据
  -> 字符表 vocab
  -> 字符转 id
  -> 随机采样 batch
  -> 输入 x 和目标 y 错开一位
  -> MiniGPT 前向传播
  -> logits
  -> Cross Entropy Loss
  -> loss.backward()
  -> optimizer.step()
  -> loss 下降
  -> 保存 checkpoint
  -> 自回归生成文本

输入和目标如何构造

字符级语言模型的训练样本是“前文预测后文”。例如：

x = "To be or no"
y = "o be or not"

x 和 y 只差一位。模型在每个位置都预测下一个字符， 这就是最小版本的大模型预训练任务。

3. 核心知识点

Embedding

字符 id 本身只是整数，没有语义。Embedding 层把每个字符 id 映射成向量： token_embedding(idx)。

Position Encoding

Self-Attention 本身不天然知道顺序，所以需要位置编码。本项目用的是可学习的位置向量： 

x = token_embedding(idx) + position_embedding(positions)

这样模型能区分“同一个字符出现在不同位置”的情况。

Self-Attention

Self-Attention 会为每个 token 计算 Query、Key、Value，然后通过 q @ k.transpose() 得到 token 之间的相关性。 

scores = q @ k.transpose(-2, -1) * (head_size ** -0.5)
weights = softmax(scores)
y = weights @ v

直观理解：当前 token 根据上下文中其他 token 的重要程度，聚合它们的信息。

Causal Mask

自回归语言模型只能看当前位置以及之前的 token。Causal Mask 用下三角矩阵挡住未来位置： 

scores = scores.masked_fill(mask == 0, -inf)

如果没有 causal mask，模型训练时会看到答案，loss 会虚假变低，生成时却不能正常工作。

Cross Entropy Loss

模型输出的是每个位置对整个 vocab 的 logits，目标是真实的下一个字符 id。 

loss = cross_entropy(
    logits.view(batch * tokens, vocab_size),
    targets.view(batch * tokens)
)

loss 越低，说明模型给真实下一个字符分配的概率越高。

反向传播和参数更新

训练真正发生在这几行：

optimizer.zero_grad(set_to_none=True)
loss.backward()
optimizer.step()
  • zero_grad 清空上一轮梯度。
  • backward 从 loss 反向计算每个参数的梯度。
  • step 用 AdamW 根据梯度更新权重。

4. 实际操作命令

训练

python.exe mini_gpt\train.py --steps 3000 --amp

--amp 表示使用 CUDA 自动混合精度，通常能减少显存占用并提升速度。

生成

python.exe mini_gpt\generate.py --checkpoint outputs\mini_gpt_shakespeare.pt --prompt "To be" --max-new-tokens 500

降低随机性

python.exe mini_gpt\generate.py --checkpoint outputs\mini_gpt_shakespeare.pt --prompt "KING:" --max-new-tokens 500 --temperature 0.7

temperature 越低越保守，越高越随机。迷你模型建议先试 0.6 到 0.9。

训练集和验证集 loss 都明显下降，说明模型确实学到了文本规律，而不是代码空跑。

6. 生成效果解读

训练 3000 step 后，示例输出类似：

To beserved your earth of heaven
May these of conceed up stand storn,
...
LADY GREY:
She a some dead of his gainstle soom her offection,

这个结果很有代表性：

  • 它学会了换行、标点、角色名格式，例如 LADY GREY:。
  • 它能生成一些真实英文词，例如 heaven、heart、lord。
  • 它也会造词，例如 conceed、storn、gainstle。

原因是它只是小型字符级模型，训练目标是预测下一个字符，不是真正理解英文语义。 但这已经足以证明 Transformer 训练链路跑通了。

7. GPU 资源观察

训练时可以用下面命令观察 GPU：

nvidia-smi -l 10

含义是每 10 秒刷新一次。也可以使用：

nvidia-smi dmon -s pucm -d 10
指标 怎么看
Memory-Usage 显存占用。看到占用上升，说明模型和 tensor 放到了 GPU。
GPU-Util GPU 计算利用率。小模型可能瞬间为 0%，因为 batch 很小、训练太快。
Power 功耗。训练更重时功耗会上升。

这次模型只有 0.83M 参数，对 RTX 4060 来说很轻，所以“用上 GPU”和“GPU 满载”不是一回事。

8. 下一步实验

继续训练

python.exe mini_gpt\train.py --steps 10000 --amp

加大模型

python.exe mini_gpt\train.py --n-layer 6 --n-head 6 --n-embd 192 --batch-size 48 --steps 8000 --amp

观察过拟合

如果 train_loss 持续下降，但 val_loss 不降反升， 说明模型开始记忆训练集而不是泛化，这就是过拟合。

建议掌握的训练知识点清单

  • 数据集切分：train / validation。
  • Tokenization：字符级、BPE、SentencePiece 的区别。
  • 模型结构：Embedding、Attention、MLP、LayerNorm、Residual。
  • 训练目标：next-token prediction。
  • 损失函数：Cross Entropy。
  • 优化器：SGD、Adam、AdamW。
  • 学习率：过大会震荡，过小会收敛慢。
  • Batch size：影响速度、显存和梯度稳定性。
  • 混合精度：FP16/BF16/AMP。
  • Checkpoint：保存模型权重、配置、词表。
  • 评估：train loss、val loss、生成样例。
  • 推理生成：temperature、top-k、top-p、自回归采样。
